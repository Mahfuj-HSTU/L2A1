1. Difference betwwen type and interface

  - interface এ extends ব্যাবহার করে টাইপ বাড়ানো যায়। এর ফলে যাকে এক্সটেন্ড করা হয় তার প্রোপার্টি গুলাও যে এক্সটন্ড করে সে পেয়ে যায়।
  যেমনঃ
  interface A {
  x:number
  }
  interface B extends A {
  y: number;
  }
  এখানে B অটোমেটিক্যালি A এর সব প্রপার্টি পেয়ে যাবে।

  - type দিয়ে যেকোনো সিঙ্গেল টাইপ ডিকলার করা গেলেও interface দিয়ে তা করা যায় না। interface এর গঠন হচ্ছে একটা object, তাই এটা অবজেক্ট টাইপ ডিফাইন করতে বেশি ব্যবহার হয়।
  অর্থাৎ type Union type সাপোর্ট করলেও, interface union type সাপোর্ট করে না। 
    
  - একই নামে interface ডিকলার করলে তা অটো মার্জ করে নিয়ে একটা টাইপ করে নেয়। type ব্যবহার করে একই নামে টাইপ ডিকলার করলে তা ইরোর দেয়।

2. Use of the keyof keyword in TypeScript
  - keyof কোনো অবজেক্ট থেকে সকল ইউনিক টাইপ পেতে ব্যাবহার করা হয়। এটা কোন টাইপের প্রোপার্টি নাম গুলোকে ইউনিক টাইপ হিসেবে বের করে আনে।
  - ডাইনামিক প্রোপার্টি আপডেটের জন্য keyof ব্যাবহার করা হয়।

3. Difference between any, unknown, and never types in TypeScript
  - any মানে কোন টাইপ নাই, যা দিবো তাই নিবো। টাইপ ব্যাবহার করা আর না করা প্রায় একই মতন। যেকোনো অপারেশন চালানো যায় কোন টাইপ চেক ছাড়া। 
  - Unknown হচ্ছে any র থেকে একটু বেটার ভার্সন। এখানে কোন অপারেশন পারফর্ম করতে হলে আগে টাইপ চেক দিতে হয়, যেন আমি যদি toUpperCase() ব্যাবহার করতে চাই তাহলে আগে সেটা স্ট্রিং টাইপ কিনা চেক করে নিতে হবে তাছাড়া কাজ করবে না, ইরোর দিবে।
  - যে ফাংশন কোন কিছু রিটার্ন করে না অথবা যে ভ্যারিয়েবলের কোন ভ্যালু নাই তার টাইপ হয় never.
